# Command: /stdd:outside-in

## Usage
```
chaos outside-in init             # Initialize outside-in TDD
chaos outside-in scaffold [change] # Generate test scaffold
chaos outside-in status           # Check status
```

## Description
Outside-in TDD approach starting from E2E tests, then integration tests, finally unit tests.

## Execution Flow
1. Define E2E test scenarios
2. Generate integration tests
3. Generate unit tests
4. Create test hierarchy

## Output
- E2E test skeleton
- Integration test skeleton
- Unit test skeleton
