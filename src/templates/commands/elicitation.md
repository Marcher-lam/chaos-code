# Command: /stdd:elicitation

## Usage
```
stdd elicitation <topic>         # Elicit requirements
stdd elicitation --method=user-story
```

## Description
Multi-perspective requirement elicitation using various techniques (user stories, scenarios, interviews).

## Execution Flow
1. Parse topic
2. Select elicitation method
3. Generate questions and prompts
4. Capture requirements

## Output
- Elicited requirements
- Question log
- Gaps identified
